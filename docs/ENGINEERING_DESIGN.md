# 工程设计文档

## 1. 工程概述

本工程为数字电源框架项目，当前包含三个目标平台工程：

| 目录 | 说明 |
|------|------|
| `hc32f334/` | HDSC HC32F334 AC/DC 平台 |
| `gd32g553c/` | GigaDevice GD32G553C 平台 |
| `plecs/` | PLECS 仿真模型 |

共享代码位于 `code/` 目录，各平台工程通过相对路径引用。

## 2. 目录结构

```
code/
├── app/           应用层（PFC、LLC、逆变器、AC 等）
├── comm/          通信模块
├── ctrl/          控制算法（PFC、逆变器、BB 等）
├── dbg/           调试与监控模块
├── interface/     硬件接口层
├── lib/           控制算法库（PID、SOGI、锁相环等）
└── section/       模块注册与链接框架
```

## 3. 调试与监控模块（code/dbg/）

### 3.1 目录说明

`code/dbg/` 提供运行时调试、性能监控、数据录波、指令追踪和命令行交互功能。每个子模块分为两层：

- **内核层**（`.c/.h`）：纯算法和数据存储，不依赖通信或打印设施
- **服务层**（`_service.c/.h`）：协议解析、shell 注册、二进制上报等，依赖内核层和通信框架

### 3.2 模块清单

| 模块 | 内核 | 服务层 | 职责 |
|------|------|--------|------|
| shell | `shell.c/h` | `shell_service.c/h` | 命令行变量注册、命令匹配、表达式求值 |
| scope | `scope.c/h` | `scope_service.c/h` | 多通道数据录波、触发、采样数据二进制上报 |
| trace | `trace.c/h` | `trace_service.c/h` | 执行追踪 FIFO、追踪记录打印与二进制上报 |
| perf | `perf.c/h` | `perf_service.c/h` | 任务/中断运行时间统计、CPU 负载计算、性能数据上报 |
| record_dict | `N/A` | 内嵌于 perf | 性能记录字典，用于 ID 分配 |

## 4. 模块注册与链接框架（code/section/）

### 4.1 目录说明

`code/section/` 提供基于链接器 section 的模块自动注册机制。通过 `SECTION_START` / `SECTION_STOP` 符号和 `AUTO_REG_SECTION` 属性，各模块在编译时自动归入对应的 section 分段，运行时不依赖动态内存分配即可完成初始化。

### 4.2 模块清单

| 文件 | 职责 |
|------|------|
| `section.c/h` | section 分段定义、注册宏、任务调度、中断服务 |
| `my_math.h` | 轻量级数学工具（pow2/3、fabs、符号判断等） |
| `platform.h` | 平台差异化宏（`CTRL_TS` 等） |

## 5. 编译配置

HC32F334 AC 平台当前使用 ARM Compiler for Embedded 6.24，优化等级 `-Oz`（极致空间优化）。
